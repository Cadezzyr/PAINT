## 部署过程：
源模型权重路径:/data/models/Qwen/Qwen3.5-27B-GPTQ-Int4

因现阶段LlamaFactory与autogptq库存在版本冲突，并不支持量化模型训练，所以安装LlamaFactory与fs-gpt两个代码库进行训练部署

1、LlamaFactory源码拉取：
`git clone https://github.com/hiyouga/LlamaFactory.git`

2、fs-gpt源码拉取：
`git clone http://git.mvpsoft.net/zh-ai/fs-gpt.git`


环境搭建：

`conda create -n ai-train python=3.11`

依赖安装：
```shell
cd fs-gpt
pip install -e .
pip install -r requirements.txt
```

从配置文件中添加数据集(路径:/data/fs-gpt/config/dataset.yaml)：
添加内容：
```
intent:
  filepath: data/alpaca_intent_recognition.jsonl
  input_field: instruction
  output_field: output
```

修改训练配置文件：
```shell
cd /data/fs-gpt/examples
vim train_sft_qlora_gptq.yaml #修改成自己的数据集，以及其他配置参数（action改为train）
```

修改数据集配置参数：
`train_dataset_names: intent(意图识别数据集)`

修改训练相关参数：
```
action: train
stage: sft
fine_tuning: lora
output_dir: models/qwen-sft-qlora-gptq
```

配置完成后开始训练：
`fs-gpt run examples/train_sft_qlora_gptq.yaml`

训练完成后得到训练好的Lora权重，路径为：/data/fs-gpt/models/qwen-sft-qlora-gptq

合并lora权重，打开合并权重配置文件：
```shell
cd /data/fs-gpt/examples
vim derive_lora.yaml 
```

修改合并相关参数：
```
action: derive
model_name_or_path: /data/models/Qwen/Qwen3.5-27B
lora_path: /data/fs-gpt-main-2026.5/models/qwen-sft-qlora-gptq
derive_method: lora
derive_dir: data/models/Qwen/Qwen3.5-27B-gptq-merge
```

配置完成后开始合并：
`fs-gpt run examples/derive_lora.yaml`

合并完成后得到合并后权重，路径为：data/models/Qwen/Qwen3.5-27B-gptq-merge

对合并后权重进行量化操作，打开模型量化配置文件：
```shell
cd /data/fs-gpt/examples
vim derive_gptq.yaml
```

修改量化相关参数：
```
action: derive
model_name_or_path: data/models/Qwen/Qwen3.5-27B-gptq-merge
derive_method: gptq
quantization_bit: 4
derive_dir: data/models/Qwen/Qwen3.5-27B-gptq-merge-Int4
```

配置完成后开始合并：
`fs-gpt run examples/derive_gptq.yaml`


最终得到两种部署方案：
1、经过 lora训练->lora权重合并->合并权重量化 三个步骤的量化版意图识别模型
2、源量化模型+lora权重 合并推理

模型部署：
修改docker compose文件：
```shell
cd /data/fs-gpt-main-2026.5/docker
vim docker-compose.yml
```

```
services:
  vllm:
    restart: unless-stopped
    build:
      context: ./service/vllm
      args:
        - VLLM_VERSION=${VLLM_VERSION}
    ports:
      - "${BIND_ADDRESS}${BIND_COLON}${VLLM_PORT}:8000"
    entrypoint: bash /etc/vllm/entrypoint.sh
    volumes:
      - /data:/data
      - ./service/vllm/etc/entrypoint.sh:/etc/vllm/entrypoint.sh
    healthcheck:
      test: [ "CMD-SHELL", "curl -sS http://127.0.0.1:8000/health || exit 1" ]
      interval: 30s
      timeout: 30s
      retries: 3
      start_period: 6m30s
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              device_ids: [ "0" ]
              capabilities: [ "gpu" ]
```

修改entrypoint文件：
`vim /data/fs-gpt-main-2026.5/docker/service/vllm/etc/entrypoint.sh`

1、经过 lora训练->lora权重合并->合并权重量化 三个步骤的量化版意图识别模型部署方式为
```
vllm serve /data/models/Qwen/Qwen3.5-27B-gptq-merge-Int4 \
--api-key fs-gpt \
--served-model-name Qwen3.5-27B-gptq-int4 \
--tensor-parallel-size 1 \
--max-model-len 8196 \
--gpu-memory-utilization 0.9 \
--max-num-seqs 8 \
--enforce-eager \
--host 0.0.0.0 \
--reasoning-parser qwen3 \
--default-chat-template-kwargs '{"enable_thinking": false}' \
--port 8000
```

2、源量化模型+lora权重 合并推理部署方式为
```
vllm serve /data/models/Qwen/Qwen3.5-27B-GPTQ-Int4 \
--api-key fs-gpt \
--served-model-name Qwen3.5-27B-GPTQ-Int4 \
--enable-lora \
--lora-modules gptq_lora=/data/fs-gpt-main-2026.5/models/qwen-sft-qlora-gptq \
--tensor-parallel-size 1 \
--max-model-len 8196 \
--gpu-memory-utilization 0.9 \
--max-num-seqs 8 \
--enforce-eager \
--host 0.0.0.0 \
--reasoning-parser qwen3 \
--default-chat-template-kwargs '{"enable_thinking": false}' \
--port 8000 
```

启动容器服务：
```shell
cd /data/fs-gpt-main-2026.5/docker
docker compose up -d
```

## 总体结论：
1、领域问题：

测试目的：验证通过lora微调学习意图识别场景的能力

结论：
针对意图识别领域问题，测试分别对源模型（source）、量化版gptq模型做lora训练的模型（lora_gptq）、bnb模型做lora训练的模型（lora_bnb）、量化版gptq模型做lora训练后的合并

模型（merge_gptq_int4）、bnb模型做lora训练后的合并模型（merge_bnb_int4）进行了测试。

最终模型输出质量的测试结论如下：merge_gptq_int4≈merge_bnb_int4≈lora_gptq＞source＞lora_bnb


2、通用问题：

测试目的：验证通过lora进行意图识别知识学习后，对原有能力的影响

结论：
对于模型通用问题，测试分别对四种模型开展测试，验证结果为lora参数的学习对各个模型的通用能力均没有显著影响。

最终模型输出质量测试结论如下：merge_gptq_int4≈merge_bnb_int4≈lora_gptq≈source≈4lora_bnb
